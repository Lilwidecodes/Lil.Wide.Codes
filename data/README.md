# Lil.Wide.Codes - Tech Portfolio Website (V1)

Lil.Wide.Codes is a versioned tech portfolio built to document real software engineering growth, 
showcase structured projects, and evolve through intentional system design rather than one-off builds.

This project is designed as a living platform - improving with skill, clarity, and technical depth over time.

---

## Purpose & Problem 

Most personal tech portfolios are: 
- visually loud but structurally weak,
- rushed without clear architecture,
- or rebuilt repeatedly without continuity.

Lil.Wide.Codes V1 solves this by:
- establishing a clean, scalable foundation,
- enforcing disciplined file organization,
- and documenting thinking, not just results.

Version 1 focuses on **structure, clarity, and correctness** before complexity.

## Project Architecture

The project follows a separation-of-concerns structure to ensure maintainability and scalability.

- `/index.html` - primary landing page
- `/pages/` - secondary pages (About, Projects, Contact, Case Studies)
- `/assests/css/` - modular stylesheets
- `layout.css`- structural layout
- `theme.css` - colors, typography, mood
- `components.css` - reusable UI components 
- `/assets/js/` - JavaScript logic
- `/assets/images/` - visual assets

This structure allows future versions to add complexity without rewriting the foundation.

## Lessons Learned

- Proper planning before coding prevents structural rewrites
- Navigation errors often come from incorrect relative paths
- Mobile-first spacing does not automatically translate well to desktop
- Modular CSS improves long-term maintainability
